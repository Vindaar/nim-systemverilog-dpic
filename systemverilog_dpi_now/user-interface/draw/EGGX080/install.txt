
  EGGX / ProCALL  version 0.80


●インストール方法

・Makefile を選んで編集

    Makefile をインストールする OS にあわせて選んでください．
  デフォルトでは Linux 用のものが設定されています．以下のよう
  にしてシンボリックリンクします．

  % ln -sf Makefile.linux Makefile  (Linux,FreeBSD)
  % ln -sf Makefile.sun Makefile    (Solaris)
  % ln -sf Makefile.netbsd Makefile (NetBSD,MacOSX)
  % ln -sf Makefile.sgi Makefile    (sgi[o32,n32,64])
  % ln -sf Makefile.hp Makefile     (HP-UX)
  % ln -sf Makefile.hp64 Makefile   (64bit HP-UX)
  % ln -sf Makefile.sun64 Makefile  (64bit Solaris)
  % ln -sf Makefile.cygwin Makefile (cygwin,X on Windows)
  % ln -sf Makefile.old Makefile    (古い OS 用)

　　Makefileの IINC, LLIB を環境にあわせて変更してください．
    64bit HP-UX, 64bit Solaris については，渡辺様が用意してく
  ださった，install_hp64_sun64.txt もあわせて御覧ください．

    EGGX のライブラリファイル，ヘッダファイルはデフォルトでは
  /usr/local/lib/, /usr/local/include/ にインストールする設定
  になっています．これは Makefile の LOCALINC LOCALLIB で指定
  します．
    root になれない場合は /usr/local/ 以下にはインストールで
  きないと思いますが，その場合はユーザのホームディレクトリ上
  の 1 つのディレクトリに EGGX のコンパイル用スクリプト，ライ
  ブラリファイル，ヘッダファイルをまとめて入れる事をお勧めし
  ます．このようにしておくと，コンパイル用スクリプトは，イン
  クルードファイル，ヘッダファイルのパスを自動的に設定するよ
  うになっていて，この場合は Makefile の LOCALINC LOCALLIB の
  修正は不要です．よく理解されている方は Makefile の LOCALINC
  LOCALLIB のある行を適宜修正してください．

    Makefile の CC, USERCC, USERFC 等の部分を確認し，必要に応
  じて修正してください．EGGX/ProCALL をビルドする時のコンパイ
  ラと,ユーザが使うコンパイラを別々に指定できます．

・make します．

・egg , eggx*.h と libeggx.a をコピーします．

　　su で root になります．libeggx.a を /usr/local/lib/ など
  のディレクトリに， eggx*.h を /usr/local/include/ などに，
  egg を /usr/local/bin/ などの パスの通ったディレクトリにコ
  ピーします． 最後に，/usr/local/bin で pro -> egg のシンボ
  リックリンクを張ります．

  [例] - root になれる場合
  # cp libeggx.a /usr/local/lib/.
  # cp eggx*.h /usr/local/include/.
  # cp egg /usr/local/bin/.
  # cd /usr/local/bin
  # ln -s egg pro

    root になれない場合は libeggx.a, eggx*.h, egg を同一ディ
  レクトリにコピーしてください．
    もちろん，Makefile の LOCALINC LOCALLIB を修正した場合は,
  適切な場所に libeggx.a, eggx*.h, egg をコピーしてください．

  [例] - root になれない場合
         (ホーム上の EGGX というディレクトリに入れる)
  % mkdir ~/EGGX
  % cp libeggx.a eggx*.h egg ~/EGGX/.
  % cd ~/EGGX/
  % ln -s egg pro
  ※コンパイルには egg か pro コマンドを使いますが，この場合
    は ~/EGGX に PATH を通すか，フルパスで実行してください．

　以上で完了です．

    コンパイルコマンド egg はシェルスクリプトなので， インス
  トールした後にエディタで修正が可能です．

・コンパイルのテスト

　　ktermなどから，
      % cd sample
　　　% egg clock.c -o clock
　としてコンパイルし，実行してみてください．
　　　% ./clock &
　　簡単なデジタル時計が現れれば，とりあえずインストールは成
  功です．


●sample ディレクトリ中のサンプルプログラム

  ◆ C 言語

  ・roche.c
    コンパイル方法
　　　% egg roche.c -o roche
    実行方法
　　　% ./roche &
    近接連星系のロッシュポテンシャルを色で、この連星系周辺で
  働く力の大きさと向き矢印で可視化します．PageUp キー(または
  Ctrl+B)と PageDown キー(または Ctrl+F )で質量比 f を変更で
  きます． 'c' キーでカラーパターンの変更，カーソルキーで色の
  レンジを変更できます． 's'キーで画像の保存，'q' キーか ESC
  キーで終了します．

  ・mandel.c
    コンパイル方法
　　　% egg mandel.c -o mandel -O2
    実行方法
　　　% ./mandel &
    有名なマンデルブロ集合の表示です．

  ・mousetest.c
    コンパイル方法
　　　% egg mousetst.c -o mousetst
    実行方法
　　　% ./mousetst &
　　
    マウス，キー入力のテストプログラムです． 実行すると 青い
  ウィンドゥが開くので，マウスでウィンドゥのどこかをクリック
  したり，キーボードから何か入力してみてください．

  ・putimg24.c
    コンパイル方法
　　　% egg putimg24.c -o putimg24
    実行方法
　　　% ./putimg24 &
    PlamoLinux のバナーを ppm ファイルから読み込んで，一括転
  送し，表示します．24bpp か 16bpp 専用です．終了方法は端末で
  Ctrl + c です．

  ・rupin.c
    コンパイル方法
　　　% egg rupin.c -o rupin
    実行方法
　　　% ./rupin &
　　
　　ルパン三世です．東風ゴシックがインストールされている必要
  があります．xtt モジュールを組み込んだ XFree86 でこれを実行
  すると X ごとクラッシュすることがありますので，注意してくだ
  さい．freetype モジュールの利用を推奨します．
    スムーズなルパン三世を見るためには，TrueType の 2 バイト
  フォントのスペーシングを -c- にする必要があります．

  ・2ch.c
    コンパイル方法
　　　% egg 2ch.c -o 2ch
    実行方法
　　　% ./2ch &
　　
    モナーとギコ猫を描くだけのプログラムです．正しく表示させ
  るには，モナーフォントが必要です．


  ◆ FORTRAN 言語(FORTRANディレクトリ)

  ・colorbar.f
    コンパイル方法
　　　% pro colorbar.f -o colorbar
    実行方法
　　　% ./colorbar &
    Version 0.71 の新機能，カラー生成サブルーチンを使用して，
  様々なカラーバーを描きます．実質，24bpp か 16bpp 専用です．
  終了すには何かキーを押します．

  ・mouse.f
    コンパイル方法
　　　% pro mouse.f -o mouse
    実行方法
　　　% ./mouse &

    マウス，キー入力のテストプログラムです． 実行すると 青い
  ウィンドゥが開くので，マウスでウィンドゥのどこかをクリック
  したり，キーボードから何か入力してみてください．

  ・fontset.f
    コンパイル方法
　　　% pro fontset.f -o fontset
    実行方法
　　　% ./fontset &

　　日本語の描画テストをするプログラムです．
